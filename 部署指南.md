# 对点咨询校招管理系统 — 快速部署指南

## 🚀 最快部署方案：Railway.app（推荐，5分钟完成）

### 为什么选择 Railway？
- ✅ 免费额度充足（500小时/月，足够小项目使用）
- ✅ 部署简单，无需信用卡（GitHub登录即可）
- ✅ 自动HTTPS，自带公网域名
- ✅ 支持 Node.js + SQLite
- ✅ 不会频繁休眠

### 部署步骤

#### 第一步：准备代码包
1. 将项目文件夹压缩为 ZIP 文件
2. 或使用 GitHub 上传代码（推荐）

#### 第二步：部署到 Railway
1. 访问 https://railway.app
2. 点击 "Login" → 选择 "GitHub" 登录（没有账号先用GitHub注册）
3. 登录后点击 "New Project"
4. 选择 "Deploy from GitHub repo"
   - 如果没有GitHub repo，选择 "Deploy from ZIP"（需要先用GitHub创建repo）
5. 连接你的GitHub账号，创建新仓库并上传代码
6. Railway 自动检测 Node.js 项目并开始部署
7. 部署完成后，点击项目 → "Settings" → "Networking"
8. 复制生成的域名（格式：`xxx.up.railway.app`）

#### 第三步：配置环境变量
在 Railway 项目页面 → "Variables" 中添加：
```
JWT_SECRET=recruitment-system-ddbd-2024-secure-key
NODE_ENV=production
```

#### 第四步：访问系统
部署成功后，通过以下链接访问：
- **候选人/大使端**：`https://你的域名.up.railway.app/`
- **HR管理端**：`https://你的域名.up.railway.app/hr.html`

## 🔄 备选方案：Render.com（完全免费）

如果 Railway 免费额度用完，可以使用 Render：

1. 访问 https://render.com
2. 注册登录（支持GitHub）
3. 点击 "New +" → "Web Service"
4. 连接GitHub仓库
5. 配置：
   - **Name**: recruitment-system
   - **Environment**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `node server.js`
6. 点击 "Create Web Service"
7. 等待部署完成（约5分钟）
8. 访问生成的域名：`https://recruitment-system.onrender.com`

⚠️ **注意**：Render 免费版在15分钟无访问后会休眠，下次访问需要等待30秒唤醒。

## 💾 数据备份（重要）

SQLite 数据库文件 `recruitment.db` 包含所有数据。

### Railway 备份方法
1. 在 Railway 项目页面 → "Deployments"
2. 点击最新部署 → "View Logs"
3. 使用 Railway CLI 下载数据库文件：
   ```bash
   railway run cat recruitment.db > backup.db
   ```

### 定期备份脚本
建议每周备份一次：
```bash
# 在本地运行，通过API触发备份（需要先在server.js中添加备份接口）
curl -H "Authorization: Bearer YOUR_TOKEN" https://你的域名/api/admin/backup
```

## 🔧 常见问题

### Q1: 部署后页面打不开？
**A**: 检查：
- Railway/Render 部署日志是否有错误
- 环境变量 `PORT` 是否设置（Railway会自动设置）
- `package.json` 中的 `start` 脚本是否正确

### Q2: 上传简历失败？
**A**: 检查：
- `uploads/` 目录是否存在
- 文件权限是否正确
- 文件大小是否超过限制（默认10MB）

### Q3: 数据库数据丢失？
**A**: Railway 重部署可能导致 SQLite 数据丢失。解决方法：
1. 在 Railway 中添加 Volume 挂载
2. 或修改代码使用外部数据库（如 PostgreSQL）

**推荐**：使用 Railway 的 PostgreSQL 插件：
- 在 Railway 项目页面 → "Add Plugin" → "PostgreSQL"
- 修改 `database.js` 使用 PostgreSQL 而非 SQLite
